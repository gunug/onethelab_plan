# 미니게임 모음형 CPI 공유 전략

## 개요

미니게임 모음 형태의 게임을 만들어, 하나의 CPI로 여러 게임(이전작 + 신작)을 플레이하게 하면 LTV를 높일 수 있을까?

---

## 기본 개념

### 기존 양산형 모델

```
[게임 A] → CPI $1.50 → LTV $2.00 → 순익 $0.50
[게임 B] → CPI $1.50 → LTV $2.00 → 순익 $0.50
[게임 C] → CPI $1.50 → LTV $2.00 → 순익 $0.50
────────────────────────────────────────────────
총 CPI: $4.50 / 총 LTV: $6.00 / 총 순익: $1.50
```

### 미니게임 모음형 모델

```
[미니게임 모음앱] → CPI $1.50 → 게임 A + B + C 플레이
                              → 총 LTV: $6.00
                              → 순익: $4.50 (3배!)
```

**핵심 아이디어:**
- 1번의 CPI로 여러 게임의 콘텐츠를 소비시킴
- 유저 획득 비용은 1번만 지불
- LTV는 모든 미니게임의 합산

### 시나리오 2: 유저 리텐션 효과 고려

**개별 게임의 문제:**
- 유저가 게임 A에 질리면 → 이탈
- 게임 B를 플레이하려면 → 새로운 CPI 필요

**모음 앱의 장점:**
- 유저가 게임 A에 질리면 → 게임 B로 전환
- 추가 CPI 없이 리텐션 유지

**리텐션 비교:**

| 지표 | 개별 게임 | 모음 앱 (예상) |
|------|----------|---------------|
| D1 리텐션 | 35% | 45% (+10%p) |
| D7 리텐션 | 10% | 20% (+10%p) |
| D30 리텐션 | 3% | 8% (+5%p) |

**리텐션 개선에 따른 LTV 변화:**

| 조건 | D30 누적 플레이 | LTV | CPI 회수율 |
|------|----------------|-----|-----------|
| 개별 게임 (D30 3%) | 45분 | $0.675 | 45% |
| 모음 앱 (D30 8%) | 120분 | $1.80 | **120%** 🟢 |

### 시나리오 3: 신작 추가 시 LTV 증가

**모음 앱의 핵심 전략: 신작 업데이트**

```
[출시 시점]
- 미니게임 5개 포함
- 유저 획득 (CPI $1.50)

[1개월 후]
- 미니게임 2개 추가 (총 7개)
- 기존 유저 재방문 유도
- 추가 CPI 없이 LTV 증가

[3개월 후]
- 미니게임 10개
- 신규 유저에게는 더 매력적인 앱
- 기존 유저의 장기 리텐션
```

**신작 추가에 따른 LTV 성장:**

| 시점 | 미니게임 수 | 예상 총 플레이 | LTV | CPI 회수율 |
|------|-----------|---------------|-----|-----------|
| 출시 | 5개 | 75분 | $1.125 | 75% |
| 1개월 | 7개 | 105분 | $1.575 | 105% 🟢 |
| 3개월 | 10개 | 150분 | $2.25 | **150%** 🟢 |
| 6개월 | 15개 | 225분 | $3.375 | **225%** 🟢 |

## 동일 예산 $7.50 투자 비교: 개별 게임 vs 모음 앱 월간 투자

**동일 조건:**
- 총 투자 예산: **$7.50**
- 광고 1회 수익: $0.015
- 미니게임 1개당 플레이: 15분
- 광고 빈도: 1분당 1회

**전략 A: 개별 게임 5개에 각각 $1.50 투자**
- 5개 게임 동시 출시
- 각 게임에 $1.50 UA 비용 지출
- 총 $7.50 한 번에 소진

**전략 B: 모음 앱에 월 $2.50씩 3개월 투자**
- 모음 앱 1개 출시 (5개 미니게임 포함)
- 1개월차: $2.50 투자 → 유저 획득
- 2개월차: $2.50 추가 투자 + 미니게임 2개 추가
- 3개월차: $2.50 추가 투자 + 미니게임 2개 추가

```

## CPI 공유의 경제학

### 1회 CPI로 여러 게임 플레이

**핵심 공식:**

```
모음 앱 효율 = (미니게임 수 × 개별 LTV) ÷ CPI
```

**예시 계산:**

| 미니게임 수 | 개별 LTV | 총 LTV | CPI | **효율** |
|------------|---------|--------|-----|----------|
| 3개 | $0.225 | $0.675 | $1.50 | 45% |
| 5개 | $0.225 | $1.125 | $1.50 | 75% |
| **7개** | $0.225 | **$1.575** | $1.50 | **105%** ⚪ 본전 |
| 10개 | $0.225 | $2.25 | $1.50 | 150% 🟢 |
| 15개 | $0.225 | $3.375 | $1.50 | 225% 🟢 |

**결론: 7개 이상의 미니게임이 있어야 본전**

### 개발 비용 고려

**개별 게임 vs 모음 앱 개발 비용:**

| 항목 | 개별 게임 5개 | 모음 앱 (5게임) |
|------|-------------|----------------|
| 게임 개발 | $10,000 × 5 = $50,000 | $10,000 × 5 = $50,000 |
| 스토어 등록 | $25 × 5 = $125 | $25 × 1 = $25 |
| 앱 관리/유지 | 5배 노력 | 1배 노력 |
| UA 캠페인 | 5개 별도 운영 | 1개만 운영 |
| **총 운영 효율** | 낮음 | **높음** |

### 크로스 프로모션 효과

**모음 앱 내 크로스 프로모션:**

```
[유저가 게임 A 플레이 중]
        ↓
"게임 B도 해보세요!" (무료)
        ↓
[게임 B 플레이]
        ↓
추가 광고 수익 (CPI 없이)
```

**개별 앱에서의 크로스 프로모션:**

```
[유저가 게임 A 플레이 중]
        ↓
"게임 B도 해보세요!" (설치 유도)
        ↓
[게임 B 설치] → CPI 발생? or 오가닉?
        ↓
전환율 낮음 (10~20%)
```

## 단점 및 리스크

| 단점 | 설명 |
|------|------|
| **앱 용량** | 여러 게임 포함 → 용량 증가 → 설치 전환율 하락 |
| **복잡성** | 개발/유지보수 복잡도 증가 |
| **스토어 노출** | 개별 게임은 각각 스토어 노출 기회 |
| **타겟팅 어려움** | "미니게임 모음"은 명확한 장르가 아님 |
| **CPI 상승 가능** | 모음 앱은 개별 하이퍼캐주얼보다 CPI 높을 수 있음 |

### CPI 상승 리스크

| 앱 유형 | 예상 CPI | 이유 |
|---------|---------|------|
| 하이퍼캐주얼 (개별) | $1.50 | 명확한 타겟, 높은 전환율 |
| 미니게임 모음 | $2.00~$2.50 | 불명확한 타겟, 낮은 전환율 |

**CPI 상승 시 손익분기점 변화:**

| CPI | 본전 미니게임 수 |
|-----|-----------------|
| $1.50 | 7개 |
| $2.00 | 9개 |
| $2.50 | 12개 |

- 10~20개 이상의 미니게임 포함

[Phase 1: 테스트]

- 유사 게임 추가 개발

[Phase 3: 성숙]
- 15~20개 미니게임

```
